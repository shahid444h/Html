HTML Forms – Complete Guide with Highlighted Code

1. Basic Form Example

HTML Code (Full Form HTML):

<form action="submit_form.php" method="post">
  <label for="name">Name:</label>
  <input type="text" id="name" name="name" required><br><br>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required><br><br>

  <input type="submit" value="Submit">
</form>

Live Output:





Explanation:
- action → URL where form data will be sent.
- method → GET or POST. POST hides data, GET shows in URL.
- required → Makes input mandatory.

1. Text Inputs (text, password, email, tel, url)

Full HTML Code:

<form action="https://httpbin.org/post" method="post" target="_blank">
  <label for="text">Text:</label>
  <input type="text" id="text" name="text" placeholder="Enter text" required><br><br>

  <label for="password">Password:</label>
  <input type="password" id="password" name="password" placeholder="Enter password" required><br><br>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" placeholder="name@example.com"><br><br>

  <label for="tel">Phone:</label>
  <input type="tel" id="tel" name="tel" placeholder="+1234567890"><br><br>

  <label for="url">Website:</label>
  <input type="url" id="url" name="url" placeholder="https://example.com"><br><br>

  <input type="submit" value="Submit">
  <input type="reset" value="Reset">
</form>

Live Output:











Explanation: Text inputs include text, password, email, tel, url. Use required, placeholder, and pattern for validation.

  <input type="range" id="range" name="range" min="0" max="10" value="5"><br><br>

  <label for="color">Favorite Color:</label>
  <input type="color" id="color" name="color" value="#ff0000"><br><br>

  <input type="submit" value="Submit">
</form>

Live Output:







Explanation: Number input allows precise numeric entry. Range input creates a slider. Color input opens a color picker. Use min, max, value attributes.

5. Textarea, Select & Datalist

Full HTML Code:

<form>
  <label for="message">Message:</label>
  <textarea id="message" name="message" placeholder="Type your message"></textarea><br><br>

  <label for="country">Country:</label>
  <select id="country" name="country">
    <option value="us">USA</option>
    <option value="uk">UK</option>
    <option value="ind">India</option>
  </select><br><br>

  <label for="browser">Browser:</label>
  <input list="browsers" id="browser" name="browser">
  <datalist id="browsers">
    <option value="Chrome">
    <option value="Firefox">
    <option value="Safari">
    <option value="Edge">
  </datalist><br><br>

  <input type="submit" value="Submit">
</form>

Live Output:







Explanation: Use textarea for multi-line input. select creates dropdown menus. datalist gives autocomplete suggestions.

8. Fieldset & Legend

Full HTML Code:

<form>
  <fieldset>
    <legend>Personal Info</legend>
    <label>Name:</label><input type="text"><br>
    <label>Email:</label><input type="email"><br>
  </fieldset>

  <fieldset>
    <legend>Account Info</legend>
    <label>Username:</label><input type="text"><br>
    <label>Password:</label><input type="password"><br>
  </fieldset;>

  <input type="submit" value="Submit">
</form>

Live Output:

Personal Info

Account Info

Explanation: fieldset groups related fields. legend gives a caption/title for the group.

  <label for="email">Email:</label>
  <input type="email" id="email" name="email"><br><br>

  <input type="submit" value="Submit via GET">
</form>

Live Output:





Explanation: GET appends data to URL as query parameters. Example URL: ?name=John&email=john@example.com. Visible in browser URL.

</form>

Live Output:





Explanation: POST sends data in the request body. URL does not show data. Safer for sensitive info like passwords.
