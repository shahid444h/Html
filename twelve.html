HTML Accessibility: alt & ARIA – Complete Guide

1. What is Accessibility?

Accessibility ensures web content can be used by everyone, including people with disabilities. Important for screen readers, keyboard navigation, and visually impaired users.

Explanation: Accessibility makes websites inclusive. Use semantic tags, proper labels, alt text, and ARIA roles to improve usability.

2. alt Attribute for Images

The alt attribute provides descriptive text for images. Screen readers read it for visually impaired users.

HTML Code:

<img src="https://www.w3schools.com/images/w3schools_green.jpg" 
     alt="W3Schools Logo" width="100">

Live Output:

Explanation: If the image fails to load, the alt text "W3Schools Logo" is displayed. Always provide meaningful alt text for images that convey important information.

3. ARIA Landmarks

ARIA roles help assistive technologies navigate your page efficiently. Common ARIA roles:

  • role="banner" → Main header
  • role="navigation" → Navigation links
  • role="main" → Main content
  • role="complementary" → Sidebar or supplementary content
  • role="contentinfo" → Footer

HTML Code:

<header role="banner">Banner / Header</header>

<nav role="navigation">Navigation Links</nav>

<main role="main">Main content</main>

<aside role="complementary">Sidebar content</aside>

<footer role="contentinfo">Footer information</footer>

Live Output:

Banner / Header
Navigation Links: Home | About | Contact
Main content of the page.
Sidebar content or additional information
Footer information
Explanation: ARIA roles improve accessibility for screen readers. Using semantic HTML (<header>, <nav>, <main>, <aside>, <footer>) combined with ARIA ensures all users can navigate and understand page structure.
