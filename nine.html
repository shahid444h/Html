HTML Comments, Entities, and Character Encoding

1. HTML Comments

Comments are notes written inside HTML code that do not appear in the browser. They help developers:

  • Explain parts of the code
  • Temporarily disable sections of code during testing
  • Provide instructions for others reading the code

Example:

<!-- Anything inside these tags is ignored by the browser -->

This text is visible in the browser.

Output:

This text is visible in the browser.

Explanation: Anything written inside <!-- comment --> is ignored and not rendered by the browser.

2. HTML Entities

HTML entities are codes used to display reserved characters or special symbols. Browsers may misinterpret certain characters (<, >, &, ") as HTML code, so entities ensure they display correctly.

Common HTML Entities:

  • &lt; → < (less than)
  • &gt; → > (greater than)
  • &amp; → & (ampersand)
  • &quot; → " (double quote)
  • &apos; → ' (single quote)
  • &nbsp; → non-breaking space
  • &copy; → © (copyright symbol)
  • &reg; → ® (registered trademark)
  • &euro; → € (Euro symbol)

Example – Using Entities:

<p>If x &lt; y then x &amp; y are valid.</p>

Output:

If x < y then x & y are valid.

Explanation: Using < and & ensures the browser displays them as text, not HTML code.

3. Character Encoding

Character encoding defines how characters are represented in HTML. UTF-8 is the most common encoding because it supports almost all characters globally.

Always declare encoding in the <head> using:

<meta charset="UTF-8">

Explanation: Without encoding, some characters (like accented letters, emojis, or symbols) may not display correctly in the browser.

Example – Displaying Special Characters:

<p>Currency symbols: &dollar;, &euro;, &pound;</p>
<p>Emoji: 😀 😍 😜</p>

Output:

Currency symbols: $, €, £

Emoji: 😀 😍 😜

4. Best Practices

  • Always use comments to explain complex sections of code.
  • Use HTML entities for reserved characters (<, >, &, ", ').
  • Declare character encoding (UTF-8) at the top of your HTML file.
  • Use entities for special symbols like ©, ®, €, and currency symbols.
